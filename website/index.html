§1.0

Abstract

Mission brief

Parvaz is a Farsi-first Android browser tunnel for Iranian users with no technical background. A helper deploys the upstream MasterHttpRelayVPN Code.gs on their own Google account; the user receives a parvaz:// link over a secure messenger (Signal or Telegram — not WhatsApp), scans a QR or taps Paste, installs a MITM certificate via Android Settings once, installs Firefox Nightly and flips one about:config flag, and taps Connect. Firefox Nightly traffic routes through Apps Script — TLS SNI www.google.com, HTTP Host script.google.com.

Scope is Firefox Nightly only. Stock Chrome and stock Firefox ignore user-installed CAs on Android 7+ and will not tunnel HTTPS. Instagram / Telegram / WhatsApp / banking apps reject user CAs the same way. These are general Android MITM constraints, not Parvaz bugs. Architecturally aligned with the proven MasterHttpRelayVPN-RUST port; Parvaz's edge is NOTAM visual identity, Farsi-by-default UI, and tighter onboarding.

Airframe specifications

Features
01Domain frontingTLS SNI says www.google.com. HTTP Host says script.google.com. DPI sees the front; Google's edge routes by the real Host internally.
02Farsi-first onboardingPaste a parvaz:// URL (forwarded via Signal or Telegram) or scan a QR → install the MITM certificate once → tap Connect. UI defaults to Persian; English is the override.
03Firefox Nightly only — honestFirefox Nightly with security.enterprise_roots.enabled = true tunnels HTTPS correctly. Stock Chrome, stock Firefox, Brave, Edge — all ignore user-installed CAs on Android 7+. Instagram, Telegram, WhatsApp, banking apps reject them too. General Android MITM limitation — pair with xray + VPS if you need those.
04SNI-rewrite fast pathGoogle-owned hosts (google.com, youtube.com, fonts.googleapis.com, …) skip Apps Script entirely and tunnel directly via SNI rewrite. No MITM, no Apps Script quota, full speed.
05Open source APKKotlin + Compose front end, embedded Go SOCKS5 sidecar. No proprietary dependencies. Ships outside the Play Store — F-Droid and sideload only.
06Zero telemetryNo analytics, no auto-update pings, no crash reporting. Your access key stays in EncryptedSharedPreferences. No third-party network calls other than the relay itself.

§5.5

Install the certificate · in Settings

Step 1 of 2 · once · ~30 seconds

In Parvaz, tap Open Settings. Parvaz drops parvaz-ca.crt into your Downloads folder and opens Android Settings. Then follow the path for your phone:

proc · cert.001 · samsung One UI 6 / Android 14
# SAMSUNG (Galaxy): Settings → Security and privacy → More security settings → Install from device storage → CA certificate → pick parvaz-ca.crt from Downloads → enter your PIN / fingerprint → tap Install anyway on the warning
proc · cert.001 · pixel / aosp Android 14 / 15
# PIXEL / stock Android: Settings → Security & privacy → More security & privacy → Encryption & credentials → Install a certificate → CA certificate → pick parvaz-ca.crt from Downloads → enter your PIN / fingerprint → tap Install anyway on the warning

Pre-requisite: your phone must have a screen lock (PIN, pattern, or password). Android refuses CA install otherwise. Can't find the menu? In Settings, search CA certificate. The first result shows the path for your phone. The warning is fine to dismiss — Parvaz's CA was generated on your phone on first launch; its private key never leaves the app's private storage.

§5.6

Install Firefox Nightly · flip one flag

Step 2 of 2 · once · ~1 minute

Stock Chrome and stock Firefox ignore the certificate you just installed — Android 7+ blocks user CAs from non-system browsers. The only no-root browser that honors user CAs is Firefox Nightly, after one flag flip.

proc · browser.001 · firefox-nightly one-time · ~1 min
# 1. Install Firefox Nightly (package id: org.mozilla.fenix) Play Store → play.google.com/store/apps/details?id=org.mozilla.fenix All channels → mozilla.org/firefox/channel/android F-Droid repo → github.com/mozilla-mobile/fenix#nightly-builds # 2. Open Firefox Nightly. In the URL bar, paste: about:config → tap to accept the warning # 3. In the search box at the top, type: security.enterprise_roots.enabled → tap the row to toggle the value to true # 4. Fully close Firefox Nightly (swipe out of recents) and reopen.

Stores blocked? Ask the helper who sent you the parvaz:// link to forward the Firefox Nightly APK via the same secure messenger (Signal or Telegram — not WhatsApp). After install, verify the package id is org.mozilla.fenix in Settings → Apps. References: Mozilla — about:config editor · Mozilla — enterprise root CAs.

§7.0

Ground support

Required · upstream

Parvaz replaces only the client half of MasterHttpRelayVPN. The Apps Script server is the upstream Code.gs unchanged — deploy it to your own Google account. If you don't need the Farsi-first NOTAM UX touches Parvaz adds, the mature MasterHttpRelayVPN-RUST port ships today with prebuilt APKs and an English + Persian walkthrough.
